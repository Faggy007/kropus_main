# Курсовая работа: форма входа и регистрации для пользователей с сохранением в базу данных

## 1. Технологии проекта

Проект построен на связке **Laravel** и **Livewire**:

- **Laravel** (^12.0) — PHP-фреймворк: маршрутизация, модели, миграции, аутентификация, сессии, валидация, события, уведомления по email.
- **Livewire** — библиотека для Laravel, дающая «живые» компоненты без отдельного JavaScript-фронтенда: форма отправляется через AJAX, состояние хранится на сервере, обновляется только нужный фрагмент страницы.
- Дополнительно в проекте используются: **Livewire Flux/Volt** (UI-компоненты), **Filament** (админ-панель), **Laravel Horizon** (очереди) и др.

Формы входа и регистрации реализованы как **Livewire-компоненты**: один класс PHP (логика + данные) и один Blade-шаблон (разметка). Отправка формы вызывает метод компонента (например, `login()` или `register()`), валидация и запись в БД выполняются на сервере.

## 2. Назначение файлов в папке Auth

Все компоненты находятся в **`app/Livewire/Auth/`** и используют общий layout **`resources/views/components/layouts/auth.blade.php`** (форма входа/регистрации без основного шапки сайта).

| Файл | Назначение |
|------|------------|
| **Login.php** | Страница входа. Свойства: `email`, `password`, `remember`. Метод `login()` — валидация, проверка лимита попыток (rate limiting), `Auth::attempt()`, при успехе — сброс счётчика, регенерация сессии, редирект на dashboard. Методы `ensureIsNotRateLimited()` и `throttleKey()` — защита от перебора паролей. |
| **Register.php** | Страница регистрации. Свойства: `name`, `email`, `password`, `password_confirmation`, `accept_privacy`. Метод `register()` — валидация (в т.ч. уникальность email и согласие с политикой), хеширование пароля, **создание пользователя в БД** через `User::create()`, событие `Registered`, авторизация и редирект на dashboard. |
| **ForgotPassword.php** | «Забыли пароль». Свойство: `email`. Метод `sendPasswordResetLink()` — валидация email, вызов `Password::sendResetLink()`, нейтральное сообщение в сессию (не раскрываем, есть ли аккаунт). |
| **ResetPassword.php** | Установка нового пароля по ссылке из письма. Свойства: `token` (из URL), `email` (из query), `password`, `password_confirmation`. В `mount()` подставляются token и email. Метод `resetPassword()` — валидация, `Password::reset()`, обновление пароля и `remember_token` в БД, событие `PasswordReset`, редирект на страницу входа. |
| **VerifyEmail.php** | Подтверждение email. Метод `sendVerification()` — повторная отправка письма со ссылкой; `logout()` — выход из аккаунта (через действие `Logout`) и редирект на главную. |
| **ConfirmPassword.php** | Подтверждение пароля перед чувствительным действием. Свойство: `password`. Метод `confirmPassword()` — проверка пароля через `Auth::guard('web')->validate()`, запись времени подтверждения в сессию (`auth.password_confirmed_at`), редирект на intended/dashboard. |
| **Account.php** | Страница «Мой аккаунт» для авторизованного пользователя. Отображает имя и email из `Auth::user()`, кнопка «Выйти» отправляет POST на маршрут `logout` (действие Logout). Layout тот же — `components.layouts.auth`. |

Шаблоны (разметка форм) лежат в **`resources/views/livewire/auth/`**: `login.blade.php`, `register.blade.php`, `forgot-password.blade.php`, `reset-password.blade.php`, `verify-email.blade.php`, `confirm-password.blade.php`, **`account.blade.php`**. Они подключаются к компонентам по соглашению имён Livewire.

## 3. Модель пользователя и создание в базе данных

### Где находится модель пользователя

- **Модель:** `app/Models/User.php`  
- Наследует `Illuminate\Foundation\Auth\User` (Authenticatable), использует трейты `HasFactory`, `Notifiable`.  
- В `$fillable`: `name`, `email`, `password`.  
- Пароль в `casts` приведён к типу `hashed` (автоматическое хеширование при присвоении).  
- Модель используется при регистрации: `User::create($validated)` в **`App\Livewire\Auth\Register`**.

### Где создаётся таблица пользователей в БД

- **Миграция:** `database/migrations/0001_01_01_000000_create_users_table.php`.  
- В методе `up()` создаются таблицы:
  - **`users`** — `id`, `name`, `email` (unique), `email_verified_at`, `password`, `remember_token`, `timestamps`.
  - **`password_reset_tokens`** — для токенов сброса пароля (email, token, created_at).
  - **`sessions`** — для хранения сессий (в т.ч. `user_id` для авторизованных пользователей).

Запись пользователя в БД появляется при **регистрации**: в компоненте **Register** после валидации вызывается `User::create($validated)` — одна строка в таблице `users` на каждого нового пользователя.

## 4. Где регистрируются маршруты аутентификации

- **Главный файл маршрутов:** `routes/web.php` — подключает группу маршрутов с middleware `frontend`, внутри неё подключается **`routes/auth.php`**.
- **Файл с маршрутами входа/регистрации:** **`routes/auth.php`**.

В нём:

- **Для гостей** (`middleware('guest')`):
  - `GET login` → компонент `Login` (имя маршрута `login`),
  - `GET register` → компонент `Register` (`register`),
  - `GET forgot-password` → компонент `ForgotPassword` (`password.request`),
  - `GET reset-password/{token}` → компонент `ResetPassword` (`password.reset`).

- **Для авторизованных** (`middleware('auth')`):
  - `GET account` → компонент `Account` (имя маршрута `account`) — страница «Мой аккаунт» с данными пользователя и кнопкой «Выйти»,
  - `GET verify-email` → компонент `VerifyEmail` (`verification.notice`),
  - `GET verify-email/{id}/{hash}` → контроллер `VerifyEmailController` (`verification.verify`),
  - `GET confirm-password` → компонент `ConfirmPassword` (`password.confirm`).

- **Выход** (доступен авторизованным):
  - `POST logout` → действие `App\Livewire\Actions\Logout` (имя маршрута `logout`).

Итого: форма входа и регистрации, сохранение пользователя в БД и все связанные сценарии (сброс пароля, подтверждение email, подтверждение пароля) реализованы на **Laravel** и **Livewire**; маршруты заданы в **`routes/auth.php`**, модель пользователя — в **`app/Models/User.php`**, создание записи в БД — в миграции **`create_users_table`** и в коде **`App\Livewire\Auth\Register`**.
